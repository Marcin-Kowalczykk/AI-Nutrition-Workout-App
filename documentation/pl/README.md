# Dokumentacja projektu

Ta dokumentacja porządkuje projekt `Training Diet App` na segmenty funkcjonalne i techniczne.
Powstała na podstawie aktualnego kodu źródłowego, routingu, endpointów API, testów oraz istniejącego `README.md`.

## Jak czytać tę dokumentację

1. Zacznij od `01-overview.md`, żeby zrozumieć produkt i jego główne segmenty.
2. Potem przejdź do `02-architecture.md`, jeśli chcesz zobaczyć architekturę aplikacji.
3. Następnie czytaj pliki segmentowe zależnie od obszaru, nad którym pracujesz.

## Spis dokumentów

- `01-overview.md` - ogólny opis produktu, zakresu i segmentów aplikacji
- `02-architecture.md` - architektura aplikacji, routing, auth, API, cache, PWA
- `03-auth-and-profile.md` - uwierzytelnianie i ustawienia profilu
- `04-diet-and-ai.md` - historia diety, skan etykiet i AI Meal Analyzer
- `05-body-measurements.md` - pomiary ciała
- `06-workout-domain.md` - historia treningów, tworzenie i edycja treningu
- `07-workout-templates.md` - szablony treningowe
- `08-exercises.md` - słownik ćwiczeń i kategorii
- `09-records-and-comparisons.md` - rekordy i porównania postępów

## Uwagi

- Folder `documentation/` zawiera dwie wersje językowe: polską i angielską.
- Istniejące materiały wdrożeniowe i SQL pozostają w `guides/`.
- Ten zestaw dokumentów jest pierwszą iteracją i powinien być uzupełniany razem z rozwojem projektu.
