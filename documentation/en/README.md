# Project Documentation

This documentation organizes `Training Diet App` into functional and technical segments.
It is based on the current source code, routing, API endpoints, tests, and the existing `README.md`.

## How to read this documentation

1. Start with `01-overview.md` to understand the product and its main segments.
2. Then move to `02-architecture.md` for the application architecture.
3. After that, read the segment files depending on the area you are working on.

## Document index

- `01-overview.md` - general product overview, scope, and application segments
- `02-architecture.md` - application architecture, routing, auth, API, cache, PWA
- `03-auth-and-profile.md` - authentication and profile settings
- `04-diet-and-ai.md` - diet history, product label scanning, and AI Meal Analyzer
- `05-body-measurements.md` - body measurements
- `06-workout-domain.md` - workout history, workout creation, and editing
- `07-workout-templates.md` - workout templates
- `08-exercises.md` - exercise and category dictionary
- `09-records-and-comparisons.md` - records and progress comparisons

## Notes

- The `documentation/` folder contains both Polish and English versions.
- Existing deployment and SQL materials remain in `guides/`.
- This is the first documentation iteration and should evolve with the project.
